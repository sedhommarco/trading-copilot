# Cycles, Sessions & Events — Calendar Overlay

> **Status: Calendar Overlay — not a strategy tab.**
> This document describes seasonal patterns and event timing that inform *when* to enter or avoid setups in the five strategy families. It is not a standalone strategy with its own watchlist. Reference it during the weekly refresh to identify upcoming seasonal tailwinds or headwinds.

**Applies to:** All 5 strategy families (especially Earnings Momentum and Macro & Volatility)
**Typical lead time:** 1–8 weeks before event

---

## Purpose

Capture predictable seasonal patterns, holiday cycles, and calendar-driven trends that create asymmetric windows for setups across any of the five strategy families. Use this overlay to:
- Add conviction to existing setups that align with a seasonal tailwind
- Avoid setups that face a seasonal headwind
- Surface new seasonal candidates to include in the weekly refresh

---

## Event categories and typical beneficiaries

| Category | Example events | Typical beneficiaries |
|---|---|---|
| Holiday cycles | Christmas (Oct–Dec), Easter, Thanksgiving, Chinese New Year | Retail (AMZN, WMT), e-commerce, logistics |
| Sports events | F1 Monaco GP, Champions League Final, Super Bowl, Olympics | Luxury (LVMH, Ferrari), sports brands (Adidas, Nike), media |
| Product launches | Apple events, Tesla Battery Day, Google I/O | AAPL, TSLA, GOOGL and ecosystem stocks |
| Seasonal cycles | "Sell in May", Q4 rally, January effect, summer doldrums | Broad indices, sector rotation |
| Crypto cycles | Bitcoin halving (next ~2028), ETF approval windows | BTC, ETH, crypto proxies (MSTR, COIN) |

---

## Integration into weekly refresh

At each refresh, check the next 4–8 weeks for:
1. Any of the above events falling within the window
2. Whether the seasonal pattern is currently intact (not overridden by macro headwinds)
3. Whether a candidate instrument already appears in another strategy tab — if so, add a seasonal note to its `rationale` rather than creating a duplicate card

Only create a new card under a strategy family (most likely Earnings Momentum or Macro & Volatility) if a seasonal setup meets the standalone entry criteria for that family.

## Key seasonal windows (Northern Hemisphere calendar)

| Month window | Seasonal theme | Typical play |
|---|---|---|
| Oct–Dec | Christmas consumer cycle | Long retail/e-commerce from early Nov; exit Dec 20–24 |
| Jan–Feb | January effect / Q4 earnings | Long growth stocks post-holiday; earnings momentum |
| Mar–Apr | Easter + Q1 earnings season | Pre-earnings setups; travel/hospitality pre-Easter |
| Apr–May pre | F1 season opener, Monaco GP | Luxury/auto pre-event rally (LVMH, Ferrari) |
| May–Jun | "Sell in May" + Champions League | Consider reducing index exposure; sports brands |
| Jun–Aug | Summer doldrums | Reduce intraday frequency; favour swing holds |
| Sep–Oct | Q3 earnings + pre-Christmas pre-positioning | Earnings Momentum re-activates; retail pre-positioning |

---

## Risk reminders

- Seasonal patterns are probabilistic, not deterministic — always require a technical or fundamental confirmation signal
- Macro regime can override seasonality entirely (e.g., a crash during Christmas season)
- Enter 2–4 weeks early; exit 1–3 days before the event peak to avoid "sell the news" risk
